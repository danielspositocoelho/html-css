HTML5 - HTML SEMÂNTICO

Uma das principais mudanças da HTML 4 para a 5 é o foco da linguagem: HTML5 é focado em sêmantica e estrutura de conteúdo, toda forma fica a cargo das CSS3.
A semântica é importante para dar sentido a cada elemento e parte do documento, ajudando os mecanismos de pesquisa a encontrar seu site com mais eficiência e também os mecanismos de acessibilidade a adaptar melhor o conteúdo.

Dessa forma, as tags da HTML4 que lidavam exclusivamente com a forma dos elementos foram trocadas por tags semânticas, ou mesmo tornadas totalmente obsoletas. Abaixo temos algumas desses casos.

Tags obsoletas e tags semânticas

As seguintes tags foram tornadas obsoletas pela HTML5

  • big

    tornava o texto grande: substituida por CSS

  • i

    inseria itálico: substituida por <em>

  • u

    inseria sublinhado:substituida por <ins>

  • b

    inseria negrito: substituida por <strong>

  • strike

    inseria texto riscado: substituida por <del>

  • blink

    fazia texto piscar: substituida por Javascript...

    Na verdade, ela é desencorajada totalmente como mau estilo!

  • marquee

    fazia o texto se mecher de determinada forma: assim como a anterior, foi substituida por Javascript Animações se fazem com JS

  • applet

    era usada para inserir um plug-in gráfico Java na página: foi substituída por <embed>

  • center

    mudava o alinhamento do texto para centralizado: substituida por CSS

  • font

    mudava diversas caracterísitcas do texto com seus atributos, como color e bgcolor: substituida por CSS

  • acronym

    adicionava abreviações: foi substituida por < abbr>

  • tt

    formatava o texto para fonte monoespaçada: foi substituída por <code>

  • xmp

    era usada para mostrar um texto preformatado, que não era interpretado como HTML e tinha sua formatação original preservada: substituida por <pre>

  • dir

    era usada para listas de diretório: foi substituída por <ul>

Essas são somente algumas das tags semânticas usadas agora. Existem muitas outras. Abaixo temos o resumo das novas tags que substituem as obsoletas

  • strong
  • em
  • ins
  • del
  • embed
  • abbr
  • code
  • pre
  • ul

Para mais exemplos de tags obsoletas e tags semânticas que as substituem confira esse artigo da W3C

Abaixo temos algumas outras tags semânticas: elas se parecem bastante com divs, mas deixam o conteúdo dentro delas com um significado mais significativo do que somente o de uma nova "divisão"

  • <article> - marca um artigo em sua integridade, distribuível e reutilizável
  • <aside> - marca conteúdo secundário, como uma pequena definição, comentário ou terminologia separada do fluxo principal do texto
  • <details>
  • <figcaption> marca legenda de imagem
  • <figure>marca elementos de imagem
  • <footer> marca o rodapé da página, onde temos detalhes adicionais como autores, links relacionados, etc
  • <header> - cabeçalho com o principal título e elementos de navegação
  • <hgroup> - grupo de título que contem o principal e seus subtítulos
  • <main> - marca parte principal da página
  • <mark> - destaca parte extremamente importante do texto
  • <nav> - marca um menu de navegação
  • <section> - agrupar tematicamente elementos e conteúdos
  • <summary>
  • <time> - marca um texto como informação sobre data, usado com o atributo datetime="year-month-day-timezone", dá detalhes para mecanismos de pequica
  • <small> - marca textos secundários, detalhes pequenos, como letras pequenas em um contrato ou informação de copyright

Para mais exemplos de tags semânticas e HTML semântico em geral, visite este artigo freeCodeCamp - Semantic HTML5 Elements Explained

Além de tags de forma - outros atributos, procedimento e tags obsoletos

Algumas outras práticas, apesar de ainda funcionarem, são desencorajadas no HTML5, por exemplo, a tag meta e os atributos border e bgcolor, muitas vezes colocados nas tags <body> ou <img>,
o uso de aspas em citações, trocadas por <q>, para citações curtas, e <blockquote> para citações longas. Confira abaixo um trecho importante do referido artigo da W3C que exemplifica isso:

        
            Obsolete but conforming features
            Features listed in this section will trigger warnings in conformance checkers.

                In the HTML syntax, authors should not specify DOCTYPEs that get parsed as obsolete permitted DOCTYPEs.

                Authors should not specify an http-equiv attribute in the Content Language state on a meta element. The lang attribute should be used instead.
                
                Authors should not specify a border attribute on an img element. If the attribute is present, its value must be the string "0". CSS should be used instead.
                
                Authors should not specify a language attribute on a script element. If the attribute is  present, its value must be an ASCII case-insensitive match for the string "JavaScript" 
                and either the type attribute must be omitted or its value must be an ASCII case-insensitive match for the string "text/javascript". The attribute should be entirely
                omitted instead (with the value "JavaScript", it has no effect), or replaced with use of the type attribute.
                
                Authors should not specify the name attribute on a elements. If the attribute is present, its value must not be the empty string. In earlier versions of the language,
                this attribute served a similar role as the id attribute. The id attribute should be used instead.

Exemplos de formatações e práticas usando tags antigas HTML4 e novas tags semânticas HTML5

Escrevendo endereço no HTML4

        
            <head>
                <title>Meu site HTML4</title>
            </head>
            <body bgcolor="blue">
                Eu moro na <u>Rua dos Capixabas, 229 - Botafogo - RJ</u>
                <marquee behavior="" direction="right">Teste de Marquee</marquee>
            </body>

Escrevendo endereço no HTML5

        
            <!DOCTYPE html>
            <html lang="pt-br">
            <head>
                <meta charset="UTF-8">
                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                <title>Exemplo de Semântica</title>
                <style>
                    body {
                        background-color: #8080da ;
                    }
                </style>
            </head>
            <body>
                <p>
                    Eu moro na <address>Rua dos Capixabas, 229 - Botafogo - RJ</address>
                </p>
            </body>
            </html>

Citação em html4

        
                <p>
                    "Eu não sou";
                </p>

Citação em html5

        
                <p> 
                    </q>Eu não sou;</q>
                </p>
